# 海马体方舟 (HippoArk)

> 面向 ACM CHI SRC 实地田野调查的老年人口述历史采集与认知健康监测系统

## 项目简介

海马体方舟是一个专为老年人口述历史采集设计的多端协同系统，旨在通过自然语音交互方式，帮助长者回忆并记录人生经历，同时实时监测认知健康指标，并将隐性知识转化为可传承的"数字传家宝"。

### 核心特性

- **适老化语音交互**：大字号界面、语音播报 (TTS)、一键录音，降低老年人使用门槛
- **多受试者数据隔离**：支持田野调查中多位老人的独立档案管理
- **实时认知健康监测**：基于 NLP 指标 (TTR/MLU) 评估语言认知状态
- **隐性知识萃取**：三级思维链 (CoT) 将碎片化口述转化为结构化《家族法典》
- **WoZ 人工干预机制**：研究员可实时覆写方言转录，应对识别错误
- **隐私数据脱敏**：自动拦截手机号、身份证号等 PII 信息

## 快速开始

### 1. 环境要求

- Python 3.9+
- 现代浏览器 (推荐 Chrome/Edge)

### 2. 安装依赖

```bash
# 克隆项目
git clone <repository-url>
cd hippo_ark_poc

# 创建虚拟环境 (可选)
python -m venv .venv
.venv\Scripts\activate  # Windows
# source .venv/bin/activate  # Linux/macOS

# 安装依赖
pip install -r requirements.txt
```

### 3. 配置 API 密钥

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件，填入您的 SiliconFlow API 密钥
# SILICONFLOW_API_KEY=sk-your-actual-key-here
```

> 获取 API 密钥：访问 [SiliconFlow](https://siliconflow.cn/) 注册账号并创建密钥

### 4. 初始化数据库

```bash
python -c "from core.db_manager import init_db; init_db()"
```

### 5. 启动服务

需要开启 **4 个终端窗口**，分别运行：

```bash
# 终端 1：启动语音转录后台服务
python transcribe_service.py

# 终端 2：启动长者端大屏 (端口 8501)
streamlit run app_elder.py --server.port 8501

# 终端 3：启动家庭端仪表盘 (端口 8502)
streamlit run app_family.py --server.port 8502

# 终端 4：启动研究员中控台 (端口 8503)
streamlit run app_researcher.py --server.port 8503
```

### 6. 访问界面

- 长者端：http://localhost:8501
- 家庭端：http://localhost:8502
- 研究员端：http://localhost:8503

## 功能模块详解

### 长者端 (app_elder.py)

专为老年人设计的语音交互界面：

- **大字号气泡**：48px 字体，高对比度配色
- **语音播报 (TTS)**：自动朗读 AI 追问内容
- **一键录音**：简化操作流程
- **记忆收集瓶**：可视化进度激励，展示已沉淀字符数
- **安全熔断**：一键暂停功能

### 家庭端 (app_family.py)

面向家属的健康监测与资产展示面板：

- **认知趋势追踪**：实时计算 TTR (词汇丰富度)、MLU (平均语句长度)、独立词汇量
- **表达活跃度图表**：基于真实时间戳的动态折线图
- **数字传家宝**：三级结构化《家族法典》
  - 事实层：客观事件还原
  - 归因层：情绪与动机分析
  - 智慧层：人生哲理提炼

### 研究员中控台 (app_researcher.py)

田野调查核心控制面板：

- **受试者管理**：新增/切换/重命名/删除受试者档案
- **实时数据流**：监控老人语音输入与 AI 转录结果
- **快速干预策略**：
  - 晚辈求教：激发表达欲
  - 情感安抚：切断敏感话题
  - 肯定价值：正向反馈激励
- **预设追问库**：20+ 领域专家设计的追问话术
- **WoZ 人工覆写**：手动修正方言识别错误
- **资产萃取**：一键生成《家族法典》JSON

### 语音转录服务 (transcribe_service.py)

后台守护进程：

- 轮询数据库中 `pending` 状态的音频记录
- 调用 SiliconFlow SenseVoice 模型进行语音识别
- 自动执行 PII 脱敏 (手机号、身份证、门牌号)
- 空音频/误触拦截 (小于 5KB 的无效录音)

## 核心算法

### NLP 认知指标

| 指标 | 全称 | 计算方式 | 临床意义 |
|------|------|----------|----------|
| TTR | Type-Token Ratio | 独立词汇数 / 总词汇数 | 词汇丰富度，反映语言检索能力 |
| MLU | Mean Length of Utterance | 单次交互有效字数 | 语言流畅度，反映认知加工效率 |
| Unique | Unique Words | 不重复词汇总数 | 词汇储备量，长期记忆健康度 |

### 三级思维链 (CoT) 资产萃取

```
口述文本 → [事实层] → [归因层] → [智慧层] → 《家族法典》
           5W1H还原    情绪动机    人生哲理
```

## 学术背景

本项目基于以下 HCI 研究方法论设计：

- **非评判性原则** (Kuttal, 2021)：避免引导性问题，尊重受访者叙事自主性
- **创伤知情方法** (Im, 2021)：设置情感安抚熔断机制，防止二次创伤
- **视觉倾斜设计**：记忆收集瓶的善意欺骗算法，激励持续表达

## 安全与隐私

- **PII 自动脱敏**：正则拦截手机号、身份证号、门牌号
- **输入验证**：老人名称严格白名单校验，防止路径穿越
- **HTML 转义**：所有动态内容经 `html.escape` 处理
- **本地存储**：数据仅存储于本地 SQLite，不上传云端

## 常见问题

### Q: 语音转录一直显示 "pending"？

确保 `transcribe_service.py` 已在后台运行，并检查 API 密钥是否有效。

### Q: 如何添加新的受试者？

在研究员中控台侧边栏输入编号 (如 `P05_赵爷爷`)，点击激活即可。

### Q: 方言识别不准确怎么办？

使用研究员中控台的 "WoZ 人工覆写" 功能，手动修正或录制普通话翻译。
